# TodoAssist - 사용자 흐름 문서 (User Flow)

> 주요 사용 시나리오별 단계별 상호작용 정의

---

## 1. 앱 시작 플로우

### 1.1 첫 실행 (온보딩)

```
사용자 더블클릭 TodoAssist.exe
        ↓
[로딩 화면 (2초)]
  "TodoAssist 초기화 중..."
        ↓
[온보딩 모달]
┌─────────────────────────────────────┐
│  TodoAssist에 오신 것을 환영합니다! │
│                                     │
│  [ 다음 ] 버튼                      │
└─────────────────────────────────────┘
    (스킵 가능)
        ↓
[온보딩 1/3] 할 일 관리
        ↓
[온보딩 2/3] 30분 타임박스 스케줄링
        ↓
[온보딩 3/3] AI 격려 비서
        ↓
[기본 설정]
  • 근무 시간 설정 (기본: 8:30~17:30)
  • Claude API 키 입력
  • 알림 설정 동의
        ↓
[ 시작 ] 버튼
        ↓
→ S1. 대시보드 화면 (빈 상태)
```

### 1.2 재실행 (캐시 로드)

```
사용자 더블클릭 TodoAssist.exe
        ↓
[로딩 화면 (0.5초)]
  "데이터 로드 중..."
        ↓
[데이터 검증]
  ✓ 데이터베이스 체크
  ✓ 마이그레이션 확인
  ✓ 설정 로드
        ↓
→ S1. 대시보드 화면 (저장된 데이터 표시)
```

## 2. 태스크 등록 플로우

### 2.1 신규 태스크 등록

```
[S1 대시보드]
    ↓
사용자 "새 태스크" 버튼 클릭
    ↓
→ S4. 태스크 등록 모달 열림
┌─────────────────────────────────────┐
│  새 태스크                          │
├─────────────────────────────────────┤
│ 제목*: [입력창]                     │
│   "품질 리포트 작성"                │
│                                     │
│ 설명: [멀티라인 입력]               │
│   "월간 품질 현황 보고서 작성..."   │
│                                     │
│ 데드라인: [날짜 선택]               │
│   2026-03-20 16:00                  │
│                                     │
│ 카테고리: [드롭다운] ▼              │
│   "품질검사" (선택) / AI 제안 가능   │
│                                     │
│ 소요 시간: [숫자 입력]              │
│   120 분 (AI가 60분으로 추정)       │
│                                     │
│ 중요도: [슬라이더] ████░░           │
│   4/5                               │
│                                     │
│ 서브태스크 추가: [+]                │
│   □ 데이터 수집 (45분)              │
│   □ 보고서 작성 (60분)              │
│   □ 검수 및 제출 (15분)             │
│                                     │
│ [ 취소 ]  [ 저장 ]                  │
└─────────────────────────────────────┘
    ↓
사용자 필드 입력 완료
    ↓
사용자 "저장" 클릭
    ↓
[백엔드]
  1. 데이터 검증 (필수 필드 확인)
  2. AI 분류 (카테고리, 소요시간 추정)
     - Claude API 호출: "이 태스크를 분류해줘"
     - 응답: {"category": "품질검사", "estimatedMinutes": 120}
  3. 태스크 ID 생성 (UUID)
  4. 데이터베이스 INSERT
  5. 서브태스크 함께 저장
    ↓
[UI 업데이트]
  ✓ 모달 닫음
  ✓ 태스크 트리 (S3)에 추가
  ✓ 대시보드 진행률 갱신
    ↓
[AI 격려 메시지] (S6 채팅창에 표시)
  "좋은 계획이네요! 차근차근 진행하면 됩니다."
    ↓
→ S1 또는 S3으로 돌아가기
```

### 2.2 태스크 수정

```
[S3 태스크 트리] 또는 [S2 칸반 보드]
    ↓
사용자 태스크 더블클릭 또는 우클릭 → "수정"
    ↓
→ S4. 태스크 수정 모달 (기존 데이터 프리필)
    ↓
사용자 필드 수정 (데드라인 연장, 중요도 상향 등)
    ↓
사용자 "저장" 클릭
    ↓
[백엔드]
  1. 변경사항 감지 (어떤 필드가 변경되었나)
  2. 필요시 AI 재분류
  3. 데이터베이스 UPDATE
  4. 진행률 재계산 (부모 태스크)
    ↓
[UI 업데이트]
  ✓ 해당 태스크 렌더링 갱신
  ✓ 트리뷰 계층 재정렬 필요시
    ↓
[옵션] AI 피드백
  "데드라인을 연장했군요. 그래도 충분히 할 수 있어요!"
    ↓
→ 이전 화면으로 돌아가기
```

## 3. 타임박스 스케줄링 플로우

### 3.1 AI 자동 스케줄 제안

```
[S1 대시보드] → 사용자 "오늘 스케줄" 버튼 클릭
                또는 [S2 칸반] 자동 진입
    ↓
→ S2. 칸반 보드 화면
┌─────────────────────────────────────────────────────────┐
│ 오늘 스케줄 (2026-03-17 Mon)  시간: 08:30 ~ 17:30      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  [AI가 제안합니다. 확인/수정하세요]                     │
│  ┌────────────────────────────────┐                   │
│  │ 로딩 중... 최적 스케줄 생성 중 │  (2초 진행)       │
│  └────────────────────────────────┘                   │
│                                                         │
└─────────────────────────────────────────────────────────┘
    ↓
[백엔드 - Claude API 호출]
  요청:
  ```
  "다음 태스크들을 오늘 스케줄해줘:
   1. 품질 리포트 (120분, 중요도 4, 데드라인 16:00)
   2. 이메일 확인 (30분, 중요도 2)
   3. 점심 (60분, 12:00 고정)
   4. 품질 회의 (90분, 중요도 5, 14:00~15:30 고정)

   현재 시간: 08:30, 근무 시간: 08:30~17:30
   최적 배치를 JSON으로 반환"
  ```

  응답:
  ```json
  [
    {
      "taskId": "task-001",
      "startSlot": 0,
      "endSlot": 3,
      "reason": "아침에 집중력이 좋으니 어려운 작업부터"
    },
    {
      "taskId": "task-002",
      "startSlot": 4,
      "endSlot": 5,
      "reason": "짧은 작업은 아침 활동 사이 정리"
    }
  ]
  ```
    ↓
[UI 표시]
┌─────────────────────────────────────────────────────────┐
│ 오늘 스케줄 (2026-03-17 Mon)  시간: 08:30 ~ 17:30      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  08:00  ┌─────────────────────┐                       │
│  ~      │ 품질 리포트 작성     │ (AI 제안)            │
│  10:00  │ 중요도: ★★★★☆     │                      │
│         │ [편집] [삭제]       │                      │
│         └─────────────────────┘                       │
│                                                         │
│  10:00  ┌─────────────┐                               │
│  ~      │ 이메일 확인 │ (AI 제안)                    │
│  10:30  └─────────────┘                               │
│                                                         │
│  10:30  ┌─────────────────────────────┐               │
│  ~      │ [빈 공간] 자유 시간         │               │
│  12:00  │ [+ 태스크 추가]             │               │
│         └─────────────────────────────┘               │
│                                                         │
│  12:00  ┌──────────┐                                  │
│  ~      │ 점심시간 │ (고정)                          │
│  13:00  └──────────┘                                  │
│                                                         │
│  [ 스케줄 저장 ]  [ 다시 생성 ]  [ 수동 편집 ]        │
│                                                         │
└─────────────────────────────────────────────────────────┘
    ↓
사용자 옵션:
  A) "스케줄 저장" → 데이터베이스 저장, S1로 이동
  B) "다시 생성" → 새로운 AI 제안 (다른 관점)
  C) "수동 편집" → 드래그앤드롭으로 조정 (아래 참고)
```

### 3.2 수동 스케줄 편집 (드래그앤드롭)

```
[S2 칸반 보드 - AI 제안 완료 상태]
    ↓
사용자 "품질 리포트" 카드 드래그
    ↓
[시각적 피드백]
  • 카드 투명도 50% (드래깅 중)
  • 드롭 가능 영역 하이라이트 (파란 테두리)
  • 충돌 감지: "이미 예약된 시간입니다" (경고음 X)
    ↓
사용자 다른 슬롯에 드롭 (예: 10:00→11:00)
    ↓
[충돌 체크]
  ✓ 11:00~13:00 빈 공간 확인
  ✓ 다른 태스크와 겹치지 않음
    ↓
[UI 즉시 업데이트]
  • 카드 위치 변경 (애니메이션 200ms)
  • TimeBox 레코드는 아직 미저장 상태
  • [저장] 버튼 활성화 (변경사항 표시)
    ↓
사용자 "스케줄 저장" 클릭
    ↓
[백엔드]
  1. 모든 TimeBox 변경사항 데이터베이스 반영
  2. 충돌 최종 검증
  3. COMMIT
    ↓
[확인 메시지]
  "스케줄이 저장되었습니다. 화이팅!"
    ↓
→ S1 대시보드로 이동
```

## 4. 태스크 실행 플로우

### 4.1 태스크 시작

```
[S2 칸반 보드]
    ↓
사용자 예정된 태스크 클릭 (또는 "지금 시작" 버튼)
  예시: "품질 리포트 작성" (08:00~10:00)
    ↓
→ 태스크 상세 패널 열림 (사이드바 우측)
┌─────────────────────────────────────┐
│ 품질 리포트 작성                    │
├─────────────────────────────────────┤
│ 상태: PENDING → IN_PROGRESS         │
│ 진행률: 0% (수동 업데이트)          │
│                                     │
│ ⏱️ 예상: 120분 → 실제: --분       │
│                                     │
│ 📌 중요도: ★★★★☆                │
│ 📅 데드라인: 2026-03-20 16:00     │
│                                     │
│ 세부사항:                           │
│ "월간 품질 현황 보고서 작성..."    │
│                                     │
│ 서브태스크:                         │
│ ☐ 데이터 수집 (45분)               │
│ ☐ 보고서 작성 (60분)               │
│ ☐ 검수 및 제출 (15분)              │
│                                     │
│ [ 시작 ]  [ 미루기 ]               │
└─────────────────────────────────────┘
    ↓
사용자 "시작" 버튼 클릭
    ↓
[백엔드]
  1. Task 상태 업데이트: pending → in_progress
  2. 시작 시간 기록
  3. TimeBox 상태 업데이트: scheduled → in_progress
    ↓
[UI 변화]
  • 칸반 보드: 해당 카드 배경색 변경 (노란색)
  • 카드 좌측에 "진행 중" 배지 표시
  • 타이머 시작 (예상 120분)
  • 우측 사이드바: 서브태스크 완료 옵션 활성화
    ↓
[AI 격려 메시지] (S6 채팅 또는 토스트)
  "시작이 반이에요! 차근차근 진행하면 됩니다. 화이팅! 💪"
  (톤: warm, 시간: 08:00)
    ↓
[선택] 타이머 사용
  • 예상 시간(120분) 카운트다운 시작
  • 시간 종료 알림: "타임박스 끝! 잠깐 쉬어보세요."
```

### 4.2 태스크 완료

```
[S2 칸반 보드 - 진행 중]
    ↓
사용자 우측 사이드바에서 서브태스크 체크
  ☑ 데이터 수집 (완료)
  ☑ 보고서 작성 (완료)
  ☑ 검수 및 제출 (완료)
    ↓
[자동 계산]
  진행률: 100% (3/3 서브태스크 완료)
    ↓
또는 사용자 태스크 카드에서 "완료" 버튼 클릭
    ↓
[확인 대화]
┌─────────────────────────────────────┐
│ 완료 확인                           │
├─────────────────────────────────────┤
│                                     │
│ "품질 리포트 작성"을 완료했나요?   │
│                                     │
│ [ 취소 ]  [ 완료 ]                  │
└─────────────────────────────────────┘
    ↓
사용자 "완료" 버튼 클릭
    ↓
[백엔드]
  1. Task 상태: in_progress → completed
  2. 완료 시간 기록: completedAt = now()
  3. 실제 소요 시간 계산: completedAt - startedAt
  4. TimeBox 상태: in_progress → completed
  5. 부모 태스크 진행률 재계산
    ↓
[UI 변화]
  • 칸반 보드: 해당 카드 사라짐 (또는 회색 처리)
  • 대시보드 진행률 바 갱신
  • 타이머 중지
    ↓
[AI 축하 메시지 + 인사이트] (S6 또는 토스트)
  "축하합니다! 품질 리포트를 완료했어요! 🎉"
  "예상 120분 대비 실제 110분 소요 - 예상보다 10분 빨랐네요!"
  (톤: warm + professional)
    ↓
[자동 다음 태스크 제안]
  "다음은 11:00의 이메일 확인입니다. 준비되셨나요?"
    ↓
→ S2 칸반 보드로 돌아감 (다음 슬롯 하이라이트)
```

### 4.3 태스크 미루기

```
[S2 칸반 보드 - 진행 중 또는 시작 대기]
    ↓
사용자 태스크 카드의 "미루기" 버튼 클릭
  또는 30분 시간초과 시 자동 감지
    ↓
[미루기 옵션 팝업]
┌─────────────────────────────────────┐
│ 품질 리포트 작성                    │
├─────────────────────────────────────┤
│                                     │
│ 언제까지 미룰까요?                  │
│                                     │
│ ○ 30분 뒤 (09:00)                   │
│ ○ 1시간 뒤 (09:30)                  │
│ ○ 내일                              │
│ ○ 다음주                            │
│                                     │
│ [ 취소 ]  [ 미루기 ]                │
└─────────────────────────────────────┘
    ↓
사용자 선택 (예: "1시간 뒤")
    ↓
[백엔드]
  1. Task 상태: in_progress → deferred (또는 pending)
  2. 미루기 기록: 새로운 TimeBox 생성 (다음 시간 슬롯)
  3. AI에 피드백: "사용자가 10시 태스크를 10시 30분으로 미뤘음"
    ↓
[UI 변화]
  • 칸반 보드: 원래 슬롯에서 제거
  • 새 슬롯(1시간 뒤)에 카드 이동
  • 카드 배지: "미뤄진 상태" 표시 (⏸️)
    ↓
[AI 넛지 메시지] (S6 채팅, 따뜻하면서 긴박)
  "알겠어요. 1시간 뒤에 다시 시작하죠."
  "조금 긴급하니까, 1시간 뒤에는 반드시 시작해봅시다!"
  (톤: urgent but warm, 카운트다운 타이머)
    ↓
[알림 설정]
  • 1시간 뒤 시스템 알림: "품질 리포트를 시작할 시간입니다!"
  • S6 채팅에 메시지 등장
    ↓
→ S2 칸반으로 돌아감 (미뤄진 카드 확인 가능)
```

## 5. 통계 확인 플로우

### 5.1 주간 리포트 확인

```
[S1 대시보드]
    ↓
사용자 "주간 리포트" 링크 클릭 또는 메뉴 → S5
    ↓
→ S5. 통계/리포트 화면
┌─────────────────────────────────────────────────────────┐

## 7. 미루기 감지 플로우

### 7.1 자동 미루기 감지

```
[백그라운드 모니터링]

  현재 시간: 08:30
  예정된 태스크: "품질 리포트" (08:00~10:00 슬롯)
    ↓
  [시간 체크]
  실제 시작 시간: 08:15 (이전 태스크 오버런)
  현재: 08:30 → 15분 지연 감지
    ↓
  ✓ 스케줄 미루기 아직 아님 (5분 여유 범위)

  ──────────────────────────────────

  현재 시간: 08:35
  예정된 태스크: "품질 리포트" (08:00~10:00 슬롯)
    ↓
  [시간 체크]
  시작 예정: 08:00
  현재: 08:35 → 35분 지연, 여전히 상태: pending
    ↓
  ✗ 미루기 감지! (임계값: 30분 초과)
    ↓
  [액션]
  1. Task 상태: pending → deferred (자동)
  2. 알림: 시스템 토스트 + 사운드
  3. S6 채팅 메시지 추가
    ↓
  [UI 표시]
┌──────────────────────────────────────┐
│ ⚠️ 알림                              │
├──────────────────────────────────────┤
│ "품질 리포트가 35분 미뤄졌어요."     │
│ "지금 시작해봅시다! 💪"             │
│                                      │
│ [ 지금 시작 ]  [ 나중에 ]           │
└──────────────────────────────────────┘
    ↓
사용자 옵션:
  A) "지금 시작" → 상태 변경: deferred → in_progress
  B) "나중에" → 닫음 (알림은 10분마다 재발생)
```
